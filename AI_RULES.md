# AI_RULES

## Перед началом задачи

Перед любой задачей сначала читать:

- `README.md`
- `PROJECT_CONTEXT.md`
- `AI_RULES.md`
- `TASKS.md`
- `CHANGELOG.md`

## Безопасность

- Не трогать без отдельного разрешения `.env`, `*.session`, приватные ключи, seed-фразы, токены, пароли и базы с чувствительными данными.
- Не показывать секреты в ответах.
- Не хранить реальные токены в коде.
- Не добавлять `.env` в GitHub.
- Если нужен реальный токен, показывать только placeholder.

## Работа с файлами

- Не удалять файлы без разрешения пользователя.
- Старые или сомнительные файлы переносить в `archive/`, а не удалять.
- Перед крупными изменениями делать бэкап важных файлов в `backups/`.
- Не делать бэкап `.env`, `*.session`, приватных ключей, seed-фраз, токенов, паролей и файлов с чувствительными данными.
- Перед изменением важного файла кратко объяснять, что будет изменено.

## Отчет после изменений

После изменений показывать список:

- созданных файлов;
- измененных файлов;
- файлов, которые не трогались;
- того, что нужно проверить вручную.

## Архитектура и зависимости

- Не менять архитектуру проекта без явной необходимости.
- Если проект маленький, не раздувать архитектуру.
- Если проект большой, сначала предложить план реорганизации.
- Не устанавливать новые зависимости без объяснения, зачем они нужны.
- Если задача непонятна, задать уточняющий вопрос.

## GitHub token

- Не требовать отзыва токена.
- Не показывать токен.
- Не хранить токен в коде.
- Использовать `GITHUB_TOKEN` из переменной окружения.
- Если нужно объяснить пользователю настройку, писать:

```text
GITHUB_TOKEN=your_github_token_here
```
