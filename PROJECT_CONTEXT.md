# Контекст проекта

## Название проекта

Impermanent Loss V2 / V3 / V4 Calculator.

## Краткое описание проекта

Статический веб-калькулятор для объяснения непостоянных потерь в пулах V2 / Classic AMM 50/50 и концентрированной ликвидности V3/V4 на примере ETH/USDC и BTC/USDC.

## Главная цель проекта

Помочь пользователю наглядно понять, как LP-позиция в V2 отличается от обычного HODL при изменении цены актива.

## Для кого этот проект

Для частных инвесторов, DeFi-пользователей и авторов образовательных материалов, которым нужен простой визуальный калькулятор impermanent loss.

## Что проект должен делать

- принимать сумму позиции;
- делить позицию 50/50 между USDC и выбранной монетой;
- показывать количество монеты в позиции;
- считать LP-результат и HODL-результат для V2;
- считать концентрированную ликвидность V3/V4 с нижней и верхней границей диапазона;
- показывать разницу против HODL;
- учитывать пользовательскую оценку комиссий;
- давать краткие подсказки по механике V2.

## Что проект не должен делать

- не должен хранить реальные токены и ключи;
- не должен давать финансовые рекомендации;
- не должен обещать точную доходность комиссий;
- не должен автоматически совершать операции с кошельками или биржами;
- не должен требовать backend для базового запуска.

## Основной сценарий использования

Пользователь открывает страницу, выбирает ETH или BTC, вводит сумму позиции, стартовую цену, будущую цену и при желании параметры комиссий. Калькулятор показывает LP, HODL и разницу между ними.

## Как запускать проект

Открыть `index.html` напрямую или запустить:

```bash
python -m http.server 4173 --bind 127.0.0.1
```

Затем открыть `http://127.0.0.1:4173/`.

## Какие внешние сервисы используются

CoinGecko public API используется для справочной цены ETH/BTC и исторического графика цены. Эта цена не подставляется как обязательная истина, пользователь может ввести округленное или свое значение.

Для текущего статического сайта переменные окружения не обязательны.

- проект пока не имеет автоматических тестов.

## Текущее состояние

Есть рабочий статический калькулятор, опубликованный через GitHub Pages. Добавлен верхний переключатель V2 / V3/V4. V2 считает Classic AMM 50/50. V3/V4 начинается с выбора пары и fee tier, затем идет двухколоночный дашборд: слева комиссии, депозит и диапазон, справа графики позиции. V3/V4 считает концентрированную ликвидность по диапазону цен, fee tier, срок активности и оценку комиссий. Ниже результатов есть отдельное сравнение HODL vs Uniswap V3/V4, исторический график цены с периодами 1M/3M/1Y и блок “Данные Uniswap как в Poolfish” для ручной оценки комиссий через объем торгов и активную ликвидность в диапазоне.

## Важные решения

- Использовать чистые HTML/CSS/JS.
- Дефолтный пример ETH: 2000 -> 4000.
- Реальная цена CoinGecko показывается как подсказка, а не принудительно подставляется в расчет.
- Комиссии считаются отдельной оценкой, не частью формулы impermanent loss.
- V3/V4 сделан отдельной вкладкой внутри `index.html`, без добавления сборщика или backend.
- В V3/V4 комиссии считаются как пользовательская оценка годовой доходности за заданное число дней; fee tier показывается как понятный выбор уровня комиссии.
- Fee tier сам по себе не меняет формулу IL; он влияет на потенциальные комиссии, но точная доходность зависит от объема торгов и доли ликвидности.
- Poolfish полезен как модель расчета: fee tier × объем торгов × доля позиции в активной ликвидности. В статической версии эти данные вводятся вручную, без скрейпинга Poolfish и без The Graph ключа во фронтенде.
- Графика V3/V4 реализована без внешних библиотек через встроенный SVG, чтобы проект оставался статическим.

## Актуальное состояние от 2026-05-22 22:22

V3/V4-калькулятор пересобран в пошаговый сценарий. Пользовательский путь теперь такой:

1. Выбрать ETH/BTC и fee tier.
2. Ввести сумму позиции, текущую цену и диапазон.
3. Ввести будущую цену и увидеть статус позиции: внутри диапазона, выше диапазона или ниже диапазона.
4. Сравнить V3/V4 с HODL и увидеть, сколько комиссий нужно, чтобы V3/V4 не проиграл HODL.

Главная логика V3/V4:

- impermanent loss считается без комиссий;
- комиссии считаются отдельно как пользовательский прогноз APR за выбранное число дней;
- break-even показывает `neededFees`, процент от депозита и required APR;
- fee tier сам по себе не меняет формулу IL, он нужен как контекст потенциальных комиссий.

Данные:

- график истории цены берется из CoinGecko `market_chart`;
- кнопка Uniswap пробует публичный subgraph для известных ETH/USDC-пулов;
- публичный Uniswap/The Graph endpoint сейчас недоступен из окружения, поэтому для надежных pool-data нужен backend/proxy с `THE_GRAPH_API_KEY`;
- ключи нельзя хранить во фронтенде и нельзя добавлять в `.env` автоматически.
