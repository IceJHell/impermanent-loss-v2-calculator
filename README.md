# Impermanent Loss V2 / V3 / V4 Calculator

Калькулятор непостоянных потерь для пулов V2 / Classic AMM 50/50 и концентрированной ликвидности V3/V4 на примере ETH/USDC и BTC/USDC.

## Что делает проект

- считает распределение позиции 50/50 между USDC и выбранной монетой;
- показывает LP-результат против обычного HODL;
- считает отдельный сценарий V3/V4 с нижней и верхней границей диапазона;
- подтягивает справочную текущую цену ETH/BTC через CoinGecko;
- позволяет оценить влияние комиссий по сроку позиции и годовой доходности;
- объясняет типовые значения impermanent loss при росте цены.

Откройте `index.html` в браузере.

Если браузер плохо открывает локальный файл, запустите простой сервер:


Затем откройте:

```text
http://127.0.0.1:4173/
```

## Частые проблемы

**Страница не открывается из файла.**  
Запустите локальный сервер командой из раздела "Быстрый старт".

**Текущая цена ETH/BTC не загрузилась.**  
CoinGecko может временно не ответить. Введите цену вручную.

**Комиссии выглядят как отдельный расчет.**  
Это ожидаемо: impermanent loss считается по изменению цены, а комиссии добавляются отдельной оценкой.

## Как работать с AI/Codex в этом проекте

В начале новой задачи AI/Codex должен читать:

- `README.md`
- `PROJECT_CONTEXT.md`
- `AI_RULES.md`
- `TASKS.md`
- `CHANGELOG.md`

Перед крупными изменениями нужно делать бэкап важных файлов в `backups/`, не трогать `.env` и не показывать секреты в ответах.
