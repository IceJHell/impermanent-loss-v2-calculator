# Память проекта

Эти правила нужны, чтобы Codex/ИИ помнил контекст проекта после перезапуска чата, отключения света или сбоя.

## Что делать в начале новой сессии

В начале каждого нового чата в этом проекте:

1. Прочитай `PROJECT_CONTEXT.md`, если файл существует.
2. Прочитай последние записи из `CODEX_LOG.md`, если файл существует.
3. Кратко скажи пользователю:
   - что обсуждали в последний раз
   - что уже сделали
   - на чём остановились
   - какой следующий шаг

## Что делать во время работы

После каждого важного обсуждения или завершённого куска работы:

1. Добавь короткую запись в `CODEX_LOG.md`.
2. Обнови `PROJECT_CONTEXT.md`, если изменились:
   - текущая цель
   - важные решения
   - изменённые файлы
   - проблемы
   - следующие шаги

Не сохраняй весь чат дословно. Сохраняй только полезную выжимку.

## Формат записи в CODEX_LOG.md

```md
## ДАТА И ВРЕМЯ

- Пользователь попросил: ...
- Что сделали: ...
- Какие файлы изменили: ...
- Какие решения приняли: ...
- Какие проблемы остались: ...
- Следующий шаг: ...
```

## Формат PROJECT_CONTEXT.md

```md
# Контекст проекта

## Текущая цель

...

## Текущее состояние

...

## Важные решения

...

## Ключевые файлы

...

## Известные проблемы

...

## Следующие шаги

...
```

## Если пользователь спрашивает, где остановились

Прочитай `PROJECT_CONTEXT.md` и `CODEX_LOG.md`, потом кратко расскажи последнее состояние проекта.

## Почему структура разбита

- `AGENTS.md` — инструкция для ИИ, как вести память
- `PROJECT_CONTEXT.md` — текущее состояние проекта
- `CODEX_LOG.md` — журнал по датам
